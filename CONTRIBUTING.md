# Contributing to CWG Recipes

Thank you for your interest in contributing to CWG Recipes! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bugfix
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Development Setup

Please refer to the [README.md](README.md) for detailed setup instructions for local development.

## Code Style

### Backend (TypeScript/Node.js)
- Use TypeScript strict mode
- Follow ESLint configuration
- Use meaningful variable and function names
- Add comments for complex logic
- Prefer async/await over callbacks
- Handle errors appropriately

### Frontend (Vue.js/TypeScript)
- Use Composition API with `<script setup>`
- Follow Vue 3 best practices
- Use TypeScript for type safety
- Keep components focused and reusable
- Use Tailwind CSS utility classes

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb in present tense (Add, Fix, Update, etc.)
- Reference issue numbers when applicable
- Example: "Add recipe export feature (#42)"

## Pull Request Process

1. Ensure your code passes all tests
2. Update documentation if needed
3. Add a clear description of your changes
4. Link any related issues
5. Wait for review and address any feedback

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on what is best for the community
- Show empathy towards other contributors

## Questions?

Feel free to open an issue for:
- Bug reports
- Feature requests
- Documentation improvements
- General questions
